LATE APPOINTMENT AUTO-DETECTION SYSTEM
═══════════════════════════════════════════════════════════════════════════════

FEATURE OVERVIEW:
The application now automatically detects when clients are late and provides
smart UI controls to manage late appointments without manual intervention.

═══════════════════════════════════════════════════════════════════════════════
HOW IT WORKS:
═══════════════════════════════════════════════════════════════════════════════

1. GRACE PERIOD: 10 minutes
   - Every scheduled appointment has a 10-minute grace period
   - If current time > scheduled time + 10 minutes:
     → Status automatically changes to "late"
     → lateSince timestamp is recorded
     → UI re-renders with action buttons

2. AUTO-DETECTION INTERVALS:
   - Runs every 60 seconds (configurable)
   - Also runs when rendering Dashboard and Appointments tabs
   - Ensures real-time status updates without page refresh

3. STATUSES:
   ✓ scheduled  - Normal appointment waiting to start
   ✓ late       - Appointment time passed + 10 min grace (ORANGE BADGE)
   ✓ forgotten_checkin - Client arrived but we forgot to mark
   ✓ no_show    - Client didn't show up
   ✓ completed  - Service finished, payment recorded
   ✓ canceled   - Appointment cancelled by us

═══════════════════════════════════════════════════════════════════════════════
QUICK START - USING LATE APPOINTMENT CONTROLS:
═══════════════════════════════════════════════════════════════════════════════

SCENARIO 1: Client is late but arrives
─────────────────────────────────────────
1. Navigate to Dashboard or Appointments tab
2. See appointment with orange "⏰ Întârzie" badge
3. Click "A venit" button
4. Status changes to "forgotten_checkin"
5. checkInTime is recorded automatically

RESULT: You have a note that client was late but eventually came.


SCENARIO 2: Client is late, offer extra time
────────────────────────────────────────────
1. See late appointment on Dashboard
2. Click "+10 min" button
3. Note automatically adds: "(+10 min întârziere)"
4. Appointment stays "late" status
5. UI updates without page reload

RESULT: You can track how many extra minutes were offered.


SCENARIO 3: Client didn't show up
──────────────────────────────────
1. See late appointment approaching deadline
2. Click "Nu a venit" button
3. Confirm the action
4. Status changes to "no_show"
5. Appointment marked in system

RESULT: Tracks no-shows for client profile and business analytics.

GRACE PERIOD CONFIGURATION:
═══════════════════════════════════════════════════════════════════════════════

To change the 10-minute grace period, edit js/appointments.js:

Current:
    const GRACE_MINUTES = 10;

To change to 5 minutes:
    const GRACE_MINUTES = 5;

To change to 15 minutes:
    const GRACE_MINUTES = 15;

To change detection interval from 60 seconds, edit js/app.js:

Current:
    setInterval(() => {
        Appointments.autoDetectLateAppointments();
    }, 60000); // 60000 ms = 60 seconds

To change to 30 seconds:
    }, 30000); // More frequent checks
